Zoom Phone integrates with hundreds of business applications through the Zoom App Marketplace, enabling click-to-dial from any business tool, automatic call logging, screen pops with caller context, calendar presence sync, and workflow automation. This guide covers the major integration categories, setup steps for each platform, and a recommended rollout sequence so you can prioritize high-impact connections first.

Browse all available integrations at [marketplace.zoom.us](https://marketplace.zoom.us).

## Prerequisites

- Admin access to the Zoom web portal
- Admin access to the third-party applications you want to integrate
- An understanding of your organization's tech stack and primary workflows
- User requirements and workflow needs identified before configuration

## General integration setup process

Use this process for any integration in the Zoom App Marketplace.

<Steps>
  <Step title="Identify integration needs">
    Survey users on their primary tools. Identify workflow bottlenecks where manual steps could be automated. Prioritize integrations by impact on daily call volume and data quality.
  </Step>
  <Step title="Access integration settings">
    Navigate to **Advanced** > **Integration** in the Zoom web portal, or browse the Zoom App Marketplace directly at [marketplace.zoom.us](https://marketplace.zoom.us).
  </Step>
  <Step title="Install the integration">
    Click the integration name, then click **Install** or **Add**. Authenticate with the third-party app and grant the required permissions.
  </Step>
  <Step title="Configure settings">
    Map data fields between systems, set notification preferences, configure user access, and customize behavior for your organization's workflow.
  </Step>
  <Step title="Test before rolling out">
    Make test calls to verify data sync, confirm screen pops display correctly, and validate that call logging writes to the correct records. Test with a known contact first.
  </Step>
  <Step title="Train users">
    Demonstrate integration features with real use-case examples. Share documentation, provide hands-on practice, and set expectations about which features are automated vs. manual.
  </Step>
  <Step title="Monitor and optimize">
    Track adoption metrics and gather user feedback. Adjust configuration as needed and add integrations based on team requests.
  </Step>
</Steps>

## Single sign-on (SSO)

<Tip>
Configure SSO before go-live. It simplifies user onboarding and eliminates password management headaches from day one.
</Tip>

SSO centralizes authentication, enables automatic user provisioning, enhances security, and simplifies access for employees across your organization.

**Supported identity providers:**
- Okta
- Azure AD
- OneLogin
- Ping Identity
- Any SAML 2.0-compliant provider

**Setup:**

<Steps>
  <Step title="Access SSO settings">
    Navigate to **Advanced** > **Single Sign-On** in the Zoom web portal.
  </Step>
  <Step title="Configure SAML 2.0 settings">
    Enter your identity provider's SAML 2.0 configuration details, including the SSO URL, issuer ID, and signing certificate.
  </Step>
  <Step title="Enter IdP metadata">
    Enter the metadata from your identity provider. Most IdPs allow you to export this as an XML file or provide a metadata URL.
  </Step>
  <Step title="Test the SSO connection">
    Test the connection with an admin account before enabling it for all users. Verify that authentication succeeds and that the correct Zoom account is accessed.
  </Step>
  <Step title="Enable for all users">
    Once testing passes, enable SSO for the full organization.
  </Step>
</Steps>

**Best for:** Enterprises requiring centralized identity management, automated provisioning, and a single authentication layer across all business tools.

## Workflow automation

### Zapier

Connect Zoom Phone to over 5,000 apps and automate workflows without writing code.

**Features:**
- Trigger actions in other apps based on Zoom Phone call events
- Create custom integrations between Zoom Phone and tools with no native integration
- Build multi-step workflows across systems

**Example use cases:**
- Add missed calls to a Google Sheet
- Post a Slack notification when a voicemail arrives
- Log all calls to Airtable
- Send an SMS via Twilio when a call ends

**Setup:**

<Steps>
  <Step title="Create a Zapier account">
    Go to zapier.com and create an account if you don't already have one.
  </Step>
  <Step title="Find Zoom Phone in Zapier">
    Search for Zoom Phone in the Zapier app directory to see available triggers and actions.
  </Step>
  <Step title="Choose a trigger">
    Select the Zoom Phone event that starts your workflow — for example: new call, new voicemail, missed call, or call ended.
  </Step>
  <Step title="Configure the action">
    Choose the target app and configure what happens when the trigger fires.
  </Step>
  <Step title="Test and enable the Zap">
    Run a test to verify data flows correctly, then enable the Zap for production use.
  </Step>
</Steps>

**Best for:** Custom workflows, quick integrations between apps that lack a native connection, and teams without dedicated development resources.

## Best practices

- **Start with high-impact integrations.** Focus on tools users spend the most time in. CRM and collaboration platforms typically deliver the most immediate value.
- **Pilot before rolling out.** Test integrations with a small pilot group first. Identify issues and refine configuration before deploying organization-wide.
- **Document use cases.** Create simple guides showing how integrations solve real problems — "How to log calls in Salesforce" is more useful than a generic feature list.
- **Avoid over-integrating.** Too many integrations create confusion and maintenance burden. Focus on quality over quantity.
- **Keep integrations updated.** Regularly review installed integrations, remove unused ones, and update configurations as business needs change.

## Common issues

<AccordionGroup>
  <Accordion title="Integration not appearing in the app">
    Verify the integration is installed and enabled in the Zoom web portal. Check that the user has the proper permissions in both Zoom and the third-party app.
  </Accordion>
  <Accordion title="Calls not logging to CRM">
    Confirm field mapping is correct. Check that the user is authenticated in both systems. Verify that API permissions are granted for the connected account.
  </Accordion>
  <Accordion title="Screen pop not showing caller information">
    Ensure the caller ID format in Zoom matches the format used in your CRM (check for country code and formatting differences). Test with a known contact first to isolate the issue.
  </Accordion>
  <Accordion title="SSO authentication fails">
    Verify that the SAML configuration matches between Zoom and your identity provider. Check that the signing certificate is current. Test with an admin account before enabling SSO for all users.
  </Accordion>
  <Accordion title="Calendar sync not updating presence">
    Verify the user authorized calendar access. Check that time zones match between the calendar and Zoom. Ensure calendar events have the correct busy or free status set.
  </Accordion>
</AccordionGroup>
